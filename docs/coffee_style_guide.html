it is ok to omit braces when using object literals, as long as that is the only thing happening there.

 
foo =
  coffee : 'black'
  cream : 'none'
  bagels :
    cream_cheese : 'fat_free'
    toasted : 'of course'
¶

in more complex situations, this is not ok. For inline object literals which fit on one line, use braces

 
foo({'name' : 'firstobject', 'title' : 'first'},
  {'name' : 'second object', 'title' : 'first'})
¶
